Our Vision

SoundScript is an open-source micro-language for writing music like code. You describe notes, chords, dynamics, patterns, and orchestration in plain text; the engine parses your script, applies musical intelligence and playback refinement, and emits a standard MIDI file. Written in C# on .NET, it runs on Windows, macOS, and Linux — plus a WebAssembly playground that runs fully client-side in any modern browser.

What sets it apart is determinism. There is no randomness anywhere in the pipeline: the same script produces byte-identical MIDI on every run, on every platform — a guarantee verified by SHA-256 comparisons in the test suite. Even "humanization" (the subtle timing and velocity jitter that makes playback feel natural) is computed deterministically, so it reproduces exactly.

The surface is deliberately minimal: no DAW, no plugins, no server, no account. Plain text in, standard MIDI out. That makes compositions behave like software — you can version them in git, diff them, review them, and rebuild them identically anywhere.

Industrial Sound Augmentation

SoundScript is not just for songs. The same phrase engine — curves, articulations, dynamic envelopes, and timing modifiers — turns machine data into non-speech audio cues an operator can understand without looking at a screen. Because every cue is a short deterministic script, the sound for "critical alarm" is byte-identical in the test lab and on the factory floor, and the cue library lives in version control alongside the rest of the system.

The repository ships five complete, runnable industrial examples:

  • Machine state sonification — idle, running, and critical rendered as distinct blocks with contrasting curve, articulation, and dynamics (industrial-machine-state.ss).
  • Conveyor timing drift — swing, push, and pull modifiers make on-time versus drifting rhythm audibly different (industrial-conveyor-drift.ss).
  • Robotic arm motion phases — approach, grip, and release as a rising swell, accented hits, and a falling fade (industrial-robotic-arm.ss).
  • Temperature trend cues — rising, stable, and falling readings mapped to a crescendo climb, a level hold, and a decrescendo descent (industrial-temperature-trend.ss).
  • Blind operator spatial awareness — an accessibility cue set of contrasting phrases, from gentle legato to sharp staccato alerts (industrial-blind-assist.ss).

Cycle-Accurate Timbre Engine — V4.1.1

Beyond MIDI, SoundScript can render actual audio — deterministic WAV or OGG — with no neural networks and no cloud service. V4 introduced the timbre engine: you pair a MIDI file with a SoundCSS stylesheet, a small CSS-like language that describes how each speech sound should be voiced (its resonances, its noise, its attack).

V4.1 changed how that audio is built. The first version painted sound in 8-millisecond slices, each one an averaged snapshot — accurate on paper, but flat to the ear. V4.1 instead rebuilds the waveform one vibration at a time: at 440 Hz a single vibration lasts about 2.27 ms, so each slice contains several complete cycles, and the engine constructs every one individually — layering its overtones, shaping it through vowel-like resonances, adding the hiss or burst of consonants — then stitches them seamlessly together.

V4.1.1 tuned that engine for realism: overtones now taper naturally per sound rather than all ringing equally (harmonic rolloff), resonances have controllable sharpness with a slight per-cycle drift (formant filtering), noise is filtered into the frequency bands real fricatives and plosives occupy (noise shaping), and consonant attacks are sharpened (transient modeling). Every stage remains fully deterministic — rendered audio is verified stable by SHA-256 regression tests.

How SoundScript Compares

SoundScript overlaps with several categories of tools without belonging to any of them. The differences below are architectural, not marketing claims.

vs. TTS engines

Modern text-to-speech systems are probabilistic — neural models whose output can vary between versions and runs. SoundScript's text pipelines (phoneme-based melody in V3.1, word-level prosody in V5, timbre rendering in V4) are rule-based and deterministic: identical text produces byte-identical output, with no model downloads and no cloud inference.

vs. DAWs

A DAW is a GUI environment built around project files, plugins, and manual arrangement. SoundScript is code: compositions are plain-text scripts you can diff, review, and version in git, compiled to standard MIDI by a CLI or in the browser — no audio workstation installation required.

vs. MIDI tools

Typical MIDI utilities edit or transform existing MIDI event data. SoundScript is a language above MIDI: you write musical intent — phrases, patterns, chord voicings, tempo automation, orchestration — and the interpreter compiles it to MIDI, with an optional deterministic MIDI-to-audio renderer on top.

Try it now

  • 1
    Clone & build
    git clone https://github.com/dharangutti/sound-script.git
    cd sound-script && dotnet build
  • 2
    Run the CLI
    dotnet run --project src/SoundScript.Cli -- run examples/full-v2-showcase.ss
  • 3
    Or try the browser playground
    Open the Playground
    Type SoundScript, compile to MIDI, and hear it instantly — runs entirely client-side at /playground/ in any modern browser (Chrome, Edge, Firefox, Safari — desktop and mobile).
  • 4
    Compose from plain text
    dotnet run --project src/SoundScript.Cli -- compose "Twinkle twinkle little star"
    The V3.1 PhonemeComposer turns the sounds of the words into a melody — or type text into the Playground's Text-to-Melody box and press Compose from text. Text-to-melody docs →
  • 5
    Export it as editable source (V6)
    dotnet run --project src/SoundScript.Cli -- compose "Twinkle twinkle little star" --emit-ss twinkle.ss
    Add --emit-ss <path> to compose/prosody and get a human-readable .ss file alongside the MIDI — hand-tweak a note, then run it with run twinkle.ss. The Playground's compose buttons offer the same export inline. What's new in V6 →
  • 6
    Make it sing
    dotnet run --project src/SoundScript.Cli -- run examples/vocal-song.ss
    Or click the Voice preset in the Playground and press Run — the lyrics are spoken over the melody, and the downloaded MIDI carries them as karaoke lyric events. Vocal track docs →

Version History

  • V2
    Composition and production features: multi-file imports, named blocks, patterns, phrases, orchestration helpers, track metadata, tempo automation, instrument layers, deterministic humanization, and advanced chord voicing.
  • V3.1
    Text-to-melody: the deterministic PhonemeComposer engine and compose CLI verb turn plain text into melody via syllables, phonemes, and musical gestures.
  • V4
    Offline timbre synthesis: deterministic MIDI-to-WAV/OGG rendering driven by SoundCSS stylesheets, via the new render verb.
  • V4.1
    Cycle-accurate synthesis: audio is reconstructed per pitch cycle (3–10 cycles per frame) with per-cycle harmonics, formants, and noise instead of flat per-frame snapshots.
  • V4.1.1
    Timbre quality tuning: harmonic rolloff curves, formant Q and drift, band-shaped noise, sharpened transients, and cycle/frame crossfading — additive, no new modules.
  • V5
    Word-level prosody: the ProsodyComposer plans pitch top-down (phrase → word → syllable) with stress detection, adding the prosody CLI verb alongside compose.
  • V6
    Current. Editable .ss export: --emit-ss serializes composed output back to hand-editable SoundScript source that round-trips to byte-identical MIDI.

About the Project

SoundScript is an open-source (MIT), deterministic music DSL that turns simple text into professional-sounding MIDI. Imports, blocks, patterns, phrases, layers, orchestration, and V3 expressive shaping extend the v1.2 engine — intentionally minimal and designed for clarity, reproducibility, and joy.

It runs everywhere: the CLI is built on cross-platform .NET (Windows, macOS, Linux), and the playground compiles to WebAssembly, so it works in any modern browser — Chrome, Edge, Firefox, and Safari, on desktop and mobile — with no server, no account, and no installation.
