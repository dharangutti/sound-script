Our Vision

Music should be as writable, versionable, and reproducible as software. SoundScript exists to make sound a first-class engineering artifact: you describe notes, chords, dynamics, phrases, and orchestration in plain text, and the engine compiles it to a standard MIDI file — the same script producing the same output, every time, on every platform.

That determinism is the foundation for everything the project is growing into: composers who want their pieces in version control, developers who want to generate music programmatically, and industrial teams who need auditable, expressive audio cues for machines, robots, and accessibility workflows. SoundScript is not a classroom toy and not tied to any single browser — it is a general-purpose text-to-MIDI language with a cross-platform CLI (built on .NET) and a WebAssembly playground that runs in Chrome, Edge, Firefox, and Safari alike.

The long-term goal is simple: a minimal, dependable language where a few lines of text carry full musical intent — from a melody sketch to a factory-floor awareness cue — with no DAW, no plugins, and no vendor lock-in.

About the Project

SoundScript is an open-source (MIT), deterministic music DSL that turns simple text into professional-sounding MIDI. Imports, blocks, patterns, phrases, layers, orchestration, and V3 expressive shaping extend the v1.2 engine — intentionally minimal and designed for clarity, reproducibility, and joy.

It runs everywhere: the CLI is built on cross-platform .NET (Windows, macOS, Linux), and the playground compiles to WebAssembly, so it works in any modern browser — Chrome, Edge, Firefox, and Safari, on desktop and mobile — with no server, no account, and no installation.
