# Cycle-Accurate Timbre Synthesis (V4.1)

SoundScript V4.0 produced correct timbre envelopes but sounded single-pitch
because `SpectralEngine` generated timbre per 8 ms **frame**, not per waveform
**cycle**. V4.1 reconstructs audio cycle-by-cycle inside each frame — the same
way a digital-to-analog converter reconstructs a waveform from pitch periods.

## The problem (V4.0)

```
Frame (8 ms) → one averaged spectral snapshot → flat pitch perception
```

## The solution (V4.1)

```
Frame (8 ms) → 3–10 pitch cycles → harmonics → formants → noise → PCM
```

Pitch determines cycle length:

```
cycle_length_ms = 1000 / frequency_hz
A4 (440 Hz) → ~2.27 ms per cycle
8 ms frame  → ~3–4 cycles (clamped to 3–10)
```

## Pipeline inside one frame

```
TimbreFramePlan
    ↓
for each cycle in frame:
    CycleGenerator      harmonic1/2/3 synthesis
    FormantFilter       vowel resonators (stateful)
    NoiseInjector       noise-fricative + noise-plosive
    TransientModel      consonant attack shaping
    ↓
CycleStitcher           stitch cycles → frame PCM
```

## New modules

| Module | Role |
|--------|------|
| `CycleGenerator` | Fundamental + 2nd/3rd harmonic per cycle |
| `FormantFilter` | Three resonators + nasal pole per cycle |
| `NoiseInjector` | Deterministic fricative/plosive noise |
| `TransientModel` | Attack envelope per cycle |
| `CycleStitcher` | Combines cycles into frame samples |
| `CyclePlanner` | Computes 3–10 cycles per frame from pitch |

## MidiToTimbreTimeline changes

`TimbreTimeline` now includes `Frames`: a list of `TimbreFramePlan` records
with:

- `PitchHz` — fundamental frequency from MIDI
- `CycleCount` — cycles in this frame (3–10)
- `CycleLengthMs` — one period duration
- `Phoneme` — aligned phoneme label
- `Profile` — resolved timbre including cycle-level attributes

## SoundCSS cycle attributes

```css
aa {
    formant1: 700Hz;
    formant2: 1100Hz;
    harmonic1: 0.9;
    harmonic2: 0.6;
    harmonic3: 0.3;
    noise-fricative: 0.2;
    noise-plosive: 0.1;
    transient: 8ms;
}

p {
    burst: 12ms;
    noise-plosive: 0.4;
    harmonic1: 0.2;
    harmonic2: 0.1;
}
```

→ Full property reference in [soundcss.md](soundcss.md)

## Determinism

Cycle synthesis uses no randomness. Phase continuity is tracked across cycles
within a note. SHA-256 regression tests in `CycleSynthesisTests` and
`FullRenderTests` verify stable WAV output.
